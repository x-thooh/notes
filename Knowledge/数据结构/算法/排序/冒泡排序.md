# 冒泡排序（Bubble Sort）
## 概念

两两比较**相邻记录**的关键字，如果反序则交换，直到没有反序的记录为止。
错误思路:
让每一个关键字，都和它后面的每一个关键字比较，如果大则交换，这样第一位置的关键字在一次循环后一定变成最小值。

## 代码

### 基础版

```go
func bubble(arr []int) {
	n := len(arr)
	for i := 0; i < n-1; i++ {
		// 每次迭代将最大的元素冒泡到最后
		for j := 0; j < n-i-1; j++ {
			// 如果当前元素比下一个元素大，则交换它们
			if arr[j] > arr[j+1] {
				arr[j], arr[j+1] = arr[j+1], arr[j]
			}
		}
	}
}
```

#### 解析
- **外层循环 `i`**：控制总共需要进行多少轮“冒泡”。由于每轮都能确定一个最大值，$n$ 个元素的数组，只需要确定 $n-1$ 个最大值，最后一个自然就固定了。所以循环范围是 `0` 到 `n-1`。
- **内层循环 `j`**：负责具体的相邻比较。随着 `i` 的增加，数组末尾已经排好序的元素越来越多。因此，每次内层循环的终点是 `n-i-1`，成功**避免了重复比较已就绪的元素**。

### 优化版
```go
func bubble(list []uint32) []uint32 {
	var (
		tempPosition = 0
		length       = len(list) - 1
		tempLength   = length
	)
	for i := 0; i < length; i++ {
		flag := false
		for j := 0; j < tempLength; j++ {
			if list[j] < list[j+1] {
				list[j], list[j+1] = list[j+1], list[j]
				tempPosition = j
				flag = true
			}
		}
		tempLength = tempPosition
		if !flag {
			break
		}
	}
	return list
}
```

####   优化点一：`flag` 哨兵（提前结束）

- **原理**：如果某次内层循环中，**一次交换都没有发生**（`flag == false`），这意味着当前数组的所有相邻元素都已经满足了顺序要求（即已经整体有序）。
    
- **效果**：如果传入的数组本来就是有序的（如 `[5, 4, 3, 2, 1]`），基础版依然会傻傻地跑完所有循环，时间复杂度为 $O(n^2)$；而加入 `flag` 后，第一轮跑完就会直接 `break`，时间复杂度瞬间降到 **$O(n)$**。
    

#### 优化点二：`tempPosition` 动态边界（局部有序优化）

- **原理**：在内层循环中，`tempPosition = j` 记录了**最后一次发生交换的位置**。这意味着，这个位置之后的所有元素，在这一轮里都没有发生过交换。换句话说，**该位置之后的子数组已经是有序的了**。
    
- **效果**：下一轮内层循环时，直接将边界缩短到 `tempLength = tempPosition`。对于“后半段基本有序”的数组，这个优化能砍掉大量毫无意义的比较操作。
